Welcome to ARTE

Introduction

ARTE stands for "Active Rich Text Editor" and is pronounced "Arty". It is a free, open source and extensible library for implementing WYSIWYG (What You See is What You Get) editors in vanilla javascript that supports active components.

With most other editors generated content is usually passive, inserted into the editor by various means, with no interaction capability. However, in ARTE content can be active, i.e. one is able to interactive with in ways appropriate to that content, for example creating protected blocks that can be edited in their own right.

You can try this by selecting this paragraph and then clicking the custom button above.

ARTE supports the following features, all implemented with a consistent plugin model:

  1. Simple block level formatting, such headings (, and ), paragraphs () and blockquotes ().
  2. Lists, including:
    • Ordered ()
    • Unordered ()
    • Indented - implemented by selecting any list items apart from the first one. If you select the first list item it will change the current list type. To see how this works select this bullet and the one above and then click the ordered list button ().
  3. Inline styles such as italic (), bold (), underline (), text colour () and highlight colour (). You can also clear all selected styling in one go using the clear button ().
  4. Undo () and redo () buttons.
  5. Custom elements, including mentions (), links (), e.g. and arbitrary custom blocks. Examples of these are provided in the toolbar above. Trying clicking the link to edit it or add a new one using the button.

Addionally, ARTE supports shortcuts. To see how this works enter the @ character followed by the @ key again at any point in the text. You should see a popup window with a set of scrollable, filterable names that can be inserted in the text via pressing `Enter` or selecting with the mouse. Note that using `@@` means that this shortcut will be available more easily on touch devices, rather than using a control, shift or tab key for example..

Customisation

ARTE supports rapid customisation since all the toolbar buttons are implemented using javascript modules following a similar pattern. They are therefore easy to modify and use as a starting point for your own plugins.

There are several configuration options available when the editor is embedded in a web page:

  1. Choice of which plugins to implement down to the level of individual buttons in the toolbar.
  2. Automatically outline numbering headings (as has been done here).
  3. Setting the undo-redo buffer size (set to 10 here).
  4. Turning debugging on or off to help during development. Currently set to off.

Building your own version

ARTE is written in the latest version of javascript but can be transpiled to support older browsers. Webpack is used for this purpose but you can replace this with your preferred build tools.

ARTE is licenced under MIT so there are no commercial constraints to using however you like or to modifying the source code for your own purposes.

The source styling is implemented use SCSS and if you want to update this file you will need to use a SASS compiler, such as the Live Sass Compiler extension for VS Code. By default the CSS generated is minified.

But remember that with great power comes great responsibility

Saving editor contents

When the editor content is saved the data is cleaned to remove any non-supported elements or styling. Additionally, custom elements can have an optional "clean" method that is invoked automatically on save to handle any custom cleaning required. This includes compressing the custom element to create a more compact, data-oriented version for saving. On load, a custom element is expanded from this compact version and required interactions added, such as listening for an edit button click.

Click the Save button below to see how this editor content is cleaned for saving. This version is output with linebreaks before all opening tags but the original saved contents is more compact but less readable.

You can try adding non-supported elements but these will be filtered out on saving, providing not only a more compact represetnation but also adding a good level of security preventing the injection of malicious code. However, you should still follow best practice and sanitise all such output before saving to a database.
